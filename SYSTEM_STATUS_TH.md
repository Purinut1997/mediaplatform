# บันทึกสถานะระบบ MIKPURINUT Media Platform

ไฟล์นี้คือเอกสารกลางสำหรับอ่านก่อนทำงานต่อทุกครั้ง เพื่อให้เข้าใจสถานะระบบโดยไม่ต้องไล่อ่านโค้ดทั้งหมดใหม่ตั้งแต่ต้น

## ข้อตกลงการทำงานต่อจากนี้

- ก่อนแก้โค้ดหรือสร้างฟีเจอร์ใหม่ ให้กลับมาอ่านไฟล์นี้ก่อนทุกครั้ง
- เมื่อทำฟีเจอร์สำคัญเสร็จ ให้ปรับไฟล์นี้ให้ตรงกับระบบจริง
- ถ้าพบว่าข้อมูลในไฟล์นี้ไม่ตรงกับโค้ด ให้ยึดโค้ดจริงเป็นหลัก แล้วอัปเดตไฟล์นี้ทันที
- ต้องคงเครดิต `Created by MIKPURINUT` และโลโก้ระบบตาม URL ที่กำหนดไว้
- เขียนโค้ดให้ประหยัด กระชับ เร็ว ขยายต่อได้ และไม่เพิ่มระบบซ้ำซ้อนโดยไม่จำเป็น

## ภาพรวมระบบ

ระบบนี้เป็นเว็บคลังสื่อและสมาชิกสำหรับโรงเรียน/องค์กร ใช้งานบน Cloudflare Pages + Functions และใช้ Neon PostgreSQL เป็นฐานข้อมูล เป้าหมายคือให้เจ้าของระบบเพิ่มสื่อ จัดสิทธิ์สมาชิก จัดการ VIP และแก้ข้อความหน้าเว็บผ่านหลังบ้านได้ โดยไม่ต้องเข้าไปแก้โค้ดบ่อย ๆ

เทคโนโลยีหลัก:

- Frontend: Vite, React, TypeScript, Tailwind CSS
- Backend: Cloudflare Pages Functions
- Database: Neon PostgreSQL
- Deploy: GitHub เชื่อม Cloudflare Pages
- Authentication: cookie session ผ่าน API

## สิ่งที่ระบบมีแล้ว

### โครงสร้างพื้นฐาน

- เชื่อม GitHub กับ Cloudflare Pages แล้ว
- Deploy production ได้ที่ Cloudflare Pages
- เชื่อม Neon ผ่าน `DATABASE_URL` แล้ว
- มี API ตรวจฐานข้อมูลและ health check
- มี schema สำหรับ users, media, media links, categories, settings และ VIP requests

### หน้าเว็บผู้ใช้

- หน้าแรกแนว portal โรงเรียน/คลังสื่อ
- ธีมเทคโนโลยี AI / Cyber / School Operations
- รองรับ Light Mode และ Dark Mode
- มีพื้นหลังแบบ grid, particle และภาพแบรนด์
- มีเครดิต `Created by MIKPURINUT`
- ใช้โลโก้ระบบตาม URL ที่กำหนด
- รองรับมือถือและเดสก์ท็อป
- มีหน้าแสดงสื่อ, รายละเอียดสื่อ, preview และปุ่มดาวน์โหลดตามสิทธิ์

### ระบบสมาชิก

- มีหน้า login
- มีหน้า register
- มีระบบตรวจ bot แบบ mock/พื้นฐานในหน้า login และ register
- ไม่แสดง user/password ตัวอย่างใน UI แล้ว
- มี role พื้นฐาน: member, admin, superadmin
- มี access พื้นฐาน: สมาชิก, VIP

### Super Admin

- เข้าหลังบ้านผ่านบัญชี superadmin ได้
- มีหน้า dashboard หลังบ้านแบบ dark dashboard
- มีเมนูหลังบ้านหลัก:
  - Dashboard
  - จัดการสื่อ
  - สมาชิกและ VIP
  - หมวดหมู่และแท็ก
  - ลิงก์ภายนอก
  - ตั้งค่าเว็บ
- เมนูหลักเริ่มใช้งานได้แล้ว ไม่ใช่ปุ่มเปล่า

### จัดการสื่อ

- เพิ่มสื่อได้
- แก้ไขสื่อได้
- ลบสื่อได้
- บันทึกลง Neon
- ใส่ชื่อ หมวดหมู่ สิทธิ์ สถานะ ราคา หน้าปก ลิงก์ และรายละเอียดได้
- API รองรับ media links หลายรายการแล้วในระดับ backend
- หน้าเพิ่ม/แก้ไขสื่อรองรับการเพิ่มลิงก์หลายรายการแล้ว
- แต่ละลิงก์กำหนดชื่อปุ่ม/ไฟล์, ชนิดลิงก์, URL, Preview URL และสิทธิ์ของลิงก์ได้
- สถานะสื่อแบบ workflow เริ่มเพิ่มแล้ว:
  - ฉบับร่าง
  - รอตรวจสอบ
  - เผยแพร่แล้ว
  - ซ่อนชั่วคราว
  - ถูกปฏิเสธ
- มีการ normalize สถานะเก่าบางค่าให้เข้ากับสถานะใหม่
- หน้าจัดการสื่อมีตัวกรองขั้นสูงแล้ว:
  - ค้นหาจากชื่อ/หมวดหมู่/รายละเอียด/ลิงก์
  - กรองคำค้นหรือแท็กแบบข้อความ
  - กรองหมวดหมู่
  - กรองสิทธิ์
  - กรองสถานะ
  - กรองวันที่เพิ่มแบบวันนี้/7 วัน/30 วัน
  - เรียงล่าสุด, ดาวน์โหลดมากสุด, เข้าชมมากสุด, ชื่อ A-Z

### สมาชิกและ VIP

- ดูรายชื่อสมาชิกได้
- ดูคำขอ VIP ได้
- อนุมัติ/ปฏิเสธคำขอ VIP ได้
- ปรับสิทธิ์สมาชิกเป็น VIP/สมาชิกได้
- เปิด/ปิดบัญชีสมาชิกได้

### หมวดหมู่

- เพิ่มหมวดหมู่ผ่านหลังบ้านได้
- ลบหมวดหมู่ผ่านหลังบ้านได้
- หมวดหมู่หลักที่ใช้ตอนนี้ เช่น AI, AppScript, โรงเรียน, งานเอกสาร, อบรม

### ตั้งค่าเว็บ

- แก้ข้อความหน้าแรกผ่านหลังบ้านได้บางส่วน
- ตั้งค่า hero title, description, image, ปุ่มหลัก/รองได้
- ตั้งค่าเปิด/ปิดการสมัคร VIP ได้
- ตั้งค่าข้อความส่วนชำระเงิน VIP, QR, ชื่อบัญชี, เลขบัญชี และข้อความยืนยันได้
- ราคา VIP ตั้งไว้เป็น 0 และยังไม่ควรเปิดแสดงราคาจริงจนกว่าจะกำหนดภายหลัง

## สิ่งที่ทำค้างหรือยังไม่สมบูรณ์

### งานค้างสำคัญระยะสั้น

1. Dashboard ยังเป็นตัวเลขมากกว่าระบบวิเคราะห์
   - ยังไม่มีกราฟยอดดาวน์โหลดรายวัน
   - ยังไม่มีสื่อยอดนิยม 10 อันดับ
   - ยังไม่มีหมวดหมู่ยอดนิยม
   - ยังไม่มีสมาชิกใหม่รายเดือน
   - ยังไม่มีคำขอ VIP รายสัปดาห์

2. ระบบ Notification Center ยังไม่สมบูรณ์
   - ควรแจ้งคำขอ VIP ใหม่
   - แจ้งลิงก์เสีย
   - แจ้งสมาชิกใหม่
   - แจ้งดาวน์โหลดผิดปกติ
   - แจ้งปัญหาฐานข้อมูลหรือ API

3. ระบบแท็กยังเป็นเพียงตัวกรองข้อความ ยังไม่มีตาราง/ฟิลด์แท็กจริงในฐานข้อมูล
   - ถ้าต้องการแท็กจริง ควรเพิ่ม schema tags และ media_tags ภายหลัง

### งานสำคัญสำหรับใช้งานจริง

1. Activity Log
   - บันทึกว่าใครทำอะไร เมื่อไหร่
   - เช่น เพิ่มสื่อ, แก้สื่อ, ลบสื่อ, อนุมัติ VIP, ปิดบัญชี

2. Admin Permission
   - แยกสิทธิ์ admin ออกจาก superadmin
   - เช่น admin อาจจัดการสื่อได้ แต่แก้ตั้งค่าเว็บหรือจัดการ superadmin ไม่ได้

3. Backup / Restore
   - ดาวน์โหลดข้อมูลเป็น JSON/CSV
   - สำรองข้อมูลสื่อ สมาชิก หมวดหมู่ และ settings
   - มีระบบกู้คืนข้อมูลอย่างระมัดระวัง

4. Error Log
   - เก็บ API error
   - database error
   - login failed
   - permission denied
   - Telegram send failed ในอนาคต

5. System Health
   - Cloudflare: Online
   - Neon Database: Connected
   - API: OK
   - Response Time
   - Last Backup
   - Error ล่าสุด

6. Broken Link Checker
   - ตรวจ Google Drive / Sheet / YouTube / External Link
   - แจ้งว่าลิงก์เปิดได้ไหม หมดอายุไหม หรือไฟล์ถูกลบไหม

7. Maintenance Mode
   - เปิด/ปิดโหมดปิดปรับปรุงจากหลังบ้าน
   - ผู้ใช้ทั่วไปเห็นข้อความปิดปรับปรุง
   - admin/superadmin ยังเข้าได้

8. Telegram Notification
   - ส่งแจ้งเตือนเมื่อมีคำขอ VIP
   - แจ้ง error สำคัญ
   - แจ้งลิงก์เสียหรือพฤติกรรมดาวน์โหลดผิดปกติ

## ลำดับงานแนะนำต่อไป

ให้ทำตามลำดับนี้เพื่อลดการแก้ซ้ำและทำให้ระบบแข็งแรงขึ้นเร็วที่สุด:

1. เพิ่ม Activity Log แบบเรียบง่ายก่อน
2. เพิ่ม Error Log และ System Health
3. เพิ่ม Backup Export เป็น JSON/CSV
4. เพิ่ม Broken Link Checker
5. เพิ่ม Maintenance Mode
6. เพิ่มกราฟ dashboard
7. เพิ่ม Telegram Notification

## กติกาด้าน UI/UX ที่ต้องรักษา

- ใช้ภาษาไทยเป็นหลัก
- ฟอนต์ต้องอ่านง่ายและดูพรีเมียม
- ปุ่มต้องใหญ่พอกับนิ้วบนมือถือ
- ช่องกรอกต้องไม่เล็ก
- ตารางต้องไม่ทำให้หน้าเว็บล้น
- บนมือถือให้เปลี่ยนตารางเป็น card layout เมื่อเหมาะสม
- ไม่ทำหน้าให้รก
- ใช้ animation นุ่มนวลแต่ไม่หนัก
- ต้องมี empty state ที่เข้าใจง่าย
- ต้องรองรับ dark/light mode
- เครดิต MIKPURINUT ต้องเล็ก สวยงาม และไม่บังปุ่ม

## หมายเหตุสำหรับ Codex รอบต่อไป

ก่อนเริ่มทำงาน ให้ทำ 3 อย่างนี้:

1. อ่านไฟล์นี้ก่อน
2. ตรวจ `git status --short`
3. ถ้าจะทำงานต่อจากฟีเจอร์ล่าสุด ให้ดูไฟล์หลักเหล่านี้:
   - `src/App.tsx`
   - `functions/api/media/index.ts`
   - `functions/api/media/[id].ts`
   - `functions/_lib/db.ts`

สถานะล่าสุดก่อนสร้างไฟล์นี้:

- `npm run lint` ผ่าน
- `npm run build` ผ่าน
- หน้าเพิ่ม/แก้ไขสื่อรองรับหลายลิงก์แล้ว
- หน้าจัดการสื่อมีตัวกรองขั้นสูงแล้ว
- หัวหลังบ้านเปลี่ยนเป็น `MIKPURINUT Media Command Center` แล้ว
